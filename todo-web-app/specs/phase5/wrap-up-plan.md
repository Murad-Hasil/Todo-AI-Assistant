```bash
# Scan tracked files for database URL patterns
git grep -rn 'postgresql://' -- ':!*.example' ':!*.md' ':!history/'

git grep -rn 'gsk_\|sk-\|Bearer ' -- ':!*.md' ':!*.example' ':!history/'

# Scan for hardcoded passwords in YAML
```

### 1.3 Local K8s Quick Start Guide

The README already contains the K8s setup. Verify these commands are accurate:

```bash
# Step 1: Start cluster
minikube start
dapr init --kubernetes --wait

# Step 2: Create dapr-secrets
kubectl create secret generic dapr-secrets \
  --from-literal=DATABASE_URL="..." \
  --from-literal=GROQ_API_KEY="..." \
  --from-literal=BETTER_AUTH_SECRET="..." \
  --from-literal=REDPANDA_BOOTSTRAP_SERVER="..." \
  --from-literal=REDPANDA_USERNAME="..." \
  --from-literal=REDPANDA_PASSWORD="..."

# Step 3: Build images
eval $(minikube docker-env)
docker build -t todo-backend:local ./todo-web-app/backend/
docker build -t todo-frontend:local ./todo-web-app/frontend/
docker build -t todo-notification:local ./todo-web-app/services/notification/

# Step 4: Deploy
helm upgrade --install todoai ./todo-web-app/k8s/charts/todoai \
  --values ./todo-web-app/k8s/charts/todoai/secrets.values.yaml

# Step 5: Apply Dapr components
kubectl apply -f todo-web-app/k8s/dapr/

# Step 6: Verify all pods running
kubectl get pods
# Expected: backend 2/2, frontend 2/2, notification 2/2 (or 1/2 if scheduler non-blocking)

# Step 7: Access
kubectl port-forward svc/todoai-frontend 3000:3000 &
kubectl port-forward svc/todoai-backend-svc 8000:8000 &
# Open http://localhost:3000
```

**Known issue**: `reminders` Kafka topic must be manually created in Redpanda Cloud before notification events flow end-to-end. Create via Redpanda Cloud UI: topic name `reminders`, 1 partition, replication factor 1.
