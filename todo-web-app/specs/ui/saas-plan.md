## Phase 0: Research & Resolved Unknowns

### R-001: Auth Route Rename Strategy

**Decision**: Create `/login` and `/register` as new pages with full implementations. Convert existing `/sign-in` and `/sign-up` to simple `redirect()` Server Components pointing to the new routes.

**Rationale**: The spec defines `/login` and `/register`. Better Auth's internal redirect config (`callbackURL`, `signInPage`) and `middleware.ts` matcher must be updated to reference the new route names. This is a pure frontend change â€” the Backend JWT dependency and auth flow are untouched. The backwards-compat redirects prevent 404s from any hardcoded links.

**Files to update for route rename**:
- `src/middleware.ts` â€” update matcher from `/sign-in` to `/login`
- `src/lib/auth.ts` â€” update `signInPage` config if set
- `src/lib/auth-client.ts` â€” no change needed (authClient handles redirects internally)

**Alternatives considered**: (1) Keep `/sign-in`/`/sign-up` and add `/login`/`/register` as pure aliases â€” rejected because it creates two canonical paths for the same page, complicating navigation links. (2) Use Next.js `rewrites` in `next.config.js` â€” rejected as more complex than server-redirect pages.

---

### R-002: Framer Motion in Next.js App Router

**Decision**: All animated components are Client Components (`"use client"`). Shared variants are defined in `lib/animations.ts` (not a component file) and imported in each Client Component.

**Rationale**: Framer Motion requires browser APIs (`window`, RAF); it cannot run in Server Components. The existing pattern in this codebase (Server Component pages, Client Component leaves) must be preserved. Wrapping entire page trees with `"use client"` would eliminate RSC benefits.

**`prefers-reduced-motion` compliance**: Use Framer Motion's built-in `useReducedMotion()` hook. Apply it in each animated component: if `shouldReduceMotion` is true, set `duration: 0` on all transitions and remove `initial`/`animate` diffs (use `{ opacity: 1, y: 0 }` for both).

---

### R-003: Mesh Background Implementation

**Decision**: Implement mesh gradients as CSS classes defined in `globals.css` using `@layer utilities`. Reference via Tailwind's `className` prop. Do not add inline `style` attributes or extend `tailwind.config.js` with custom gradient values (to avoid bloat).

**Rationale**: FR-028 mandates Tailwind CSS only. Custom CSS in `@layer utilities` is the approved Tailwind extension pattern. Keeping gradient definitions in one place (`globals.css`) ensures consistent appearance across login and register pages.

**Implementation**:

### R-004: Sidebar State Management

**Decision**: Sidebar open/closed state lives in the `(dashboard)/layout.tsx` as a Client Component (`"use client"`). Pass `isOpen`/`setIsOpen` as props to the `Sidebar` child. The dashboard `page.tsx` remains a Server Component.

**Rationale**: The sidebar toggle is a pure UI concern local to the layout. No global state manager (Zustand, Context) is warranted for a single boolean. The layout wraps the page `children` â€” making layout a Client Component does not force page to be a Client Component (children are rendered independently).

**Escape key listener**: `useEffect` with `keydown` listener in Sidebar Client Component. Cleans up on unmount.

### R-005: `tailwind-merge` + `clsx` Usage Pattern

**Decision**: Export a `cn(...inputs)` utility from `lib/utils.ts`:
```ts
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"
export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }
```
Import `cn` in all components that build dynamic class strings.

### 1.3 Animation System (`lib/animations.ts`)

```ts
// src/lib/animations.ts
import type { Variants, Transition } from "framer-motion"

export const fadeInUp: Variants = {
  hidden: { opacity: 0, y: 24 },
  visible: { opacity: 1, y: 0 },
}

export const staggerContainer: Variants = {
  hidden: {},
  visible: {
    transition: {
      staggerChildren: 0.1,
      delayChildren: 0.05,
    },
  },
}

export const slideInRight: Variants = {
  hidden: { x: "100%", opacity: 0 },
  visible: { x: 0, opacity: 1 },
  exit: { x: "100%", opacity: 0 },
}

export const springTransition: Transition = {
  type: "spring",
  stiffness: 380,
  damping: 30,
}

export const easeTransition: Transition = {
  duration: 0.4,
  ease: [0.25, 0.1, 0.25, 1],
}

/** Returns reduced-motion-safe variants â€” all values identical (no visual diff) */
export function reducedVariants(variants: Variants): Variants {
  return Object.fromEntries(
    Object.keys(variants).map((k) => [k, { opacity: 1, x: 0, y: 0 }])
  )
}
```
