# Cloud-Native & Event-Driven Ops

## Instructions
1. Containerize applications using multi-stage Dockerfiles.
2. Create Helm charts for `frontend`, `backend`, and `notification-service`.
3. Configure Dapr components for `pubsub.kafka` and `state.postgresql`.
4. Implement Kafka topics for `task-events`, `reminders`, and `task-updates`.

## Examples
- Writing a `dapr.io/v1alpha1` Component manifest for Kafka.
- Using `kubectl-ai` or `kagent` prompts to optimize cluster resource allocation.
